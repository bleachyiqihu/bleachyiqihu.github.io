最短路径算法

发表于 2022-12-05 更新于 2022-12-07 分类于 算法 阅读次数：

最短路径算法是用来求解图中两个顶点间最小路径的。

本文介绍两种常见的最短路径算法，Floyd算法和Dijkstra算法，并附上一道和最短路径算法相关的leetcode题目。

Floyd算法

该算法使用于任何图，无向、有向、正权、负权都可以适用。唯一的限制是图必须有最短路径（无负环）。

算法实现简单，但时间复杂度较高，为$O(N^3)$

思路

我们假设图中有N个节点，每个节点的编号分别为1，2，3，…，N。构建一个数组f[k][x][y]，含义是从x出发，中间只能经过1，2，…，k这些编号的节点，走到y节点的最短路径长度。特别的，f[0][x][y]的值被这样规定：若x，y之间有边，则f[0][x][y]为x到y的边权；若x，y之间没有边，则数组值为$\infty$；若x=y，则数组值为0。

很明显，f[N][x][y]的值，就是x到y的最短路径长度。

如果我们已知f[k-1][x][y]的值，如何推出f[k][x][y]的值呢？f[k][x][y]相较f[k-1][x][y]，相当于在x到达y的最短路径选择上，多出了k这个编号的节点，我们可以选择经过k，也可以选择不经过。如果不经过k，那么x到y的路径长度依旧是f[k-1][x][y],如果经过k，那么x到y的路径长度为f[k-1][x][k]+f[k-1][k][y]。所以我们可以得到递推公式：

我们把f[0][x][y]按照规定初始化好之后，即可按上述递推公式，把f[N][x][y]求出。


for (int k = 1; k <= N; k++) {
	for (int x = 1; x <= N; x++) {
		for (int y = 1; y <= N; y++) {
			f[k][x][y] = Math.min(f[k-1][x][y], f[k-1][x][k] + f[k-1][k][y]);
		}
	}
}

空间优化

按照上述思路，Floyd算法的时间复杂度为$O(N^3)$，空间复杂度也为$O(N^3)$。但实际f[k][x][y]的第一维是不需要的，每层k变量上的迭代，原地修改f[x][y]的值即可。代码可调整成


for (int k = 1; k <= N; k++) {
	for (int x = 1; x <= N; x++) {
		for (int y = 1; y <= N; y++) {
			f[x][y] = Math.min(f[x][y], f[x][k] + f[k][y]);
		}
	}
}

我们尝试证明此结论，假设当前迭代k = K，在求解f[x][y]的过程中，若f[x][y] < f[x][K] + f[K][y]，则f[x][y]的值不变，也不会对后续求解产生影响。反之，f[x][y]会被覆盖为f[x][K] + f[K][y]，当且仅当x = K 或 y = K时，该覆盖才可能对本次迭代（k = K这层迭代）的求解产生影响，因为依赖求解的数组值的下标特点，要么第一个下标为K，要么第二个下标为K。而当x = K时，f[K][y] = Math.min(f[K][y], f[K][K] + f[K][y]) = Math.min(f[K][y], f[K][y]) = f[K][y]，同理当y = K时，f[x][K] = f[x][K]，两种情况其值都不会改变。不会对后续计算产生影响。

优化后，Floyd算法的空间复杂度下降到$O(N^2)$.

Dijkstra算法

思路

Dijkstra算法是一种解决固定出发点，求解该点到图中其他节点的最短路径的算法。Dijkstra算法只能应用于正权图中。

Dijkstra算法把图中节点分为两类，一类是已经找到最短路径的节点（记为集合S），一类是尚未找到最短路径的节点（记为集合T）。在算法开始时，集合S为空集，图中所有节点均存在于集合T中。我们定义两个函数，一个为D(n)，表示节点a到节点n的最短路径长度；另一个为标记函数，表示从a出发只包含S集合中节点，走到n节点的最短路径长度，k为算法迭代次数。被初始化为0，其余节点的dis函数值，被初始化为$\infty$。

Dijkstra算法需重复两个步骤：

  1. 从集合T中选取dis函数值最小的节点，将其加入集合S；
  2. 和最新加入集合S的节点邻接的所有节点，更新其dis函数的值。节点x和y的边权若记为w(x, y)，设最新加入集合S的节点为m，则更新的公式为

证明

下面证明Dijkstra算法的正确性。

设a为起点，若证明，在第k次迭代：

  • 加入S中的节点v($v \neq a$)的标记是从a出发到v的最短路径长度
  • 不在S中的顶点标记是只包含S中顶点（不包含该节点）的从a到该顶点的最短路径长度

用数学归纳法证明，当迭代数k = 0时，S为空集，a的标记值为0，其余节点的标记值为$\infty$，满足上述两点。

设第k次迭代时，上述假设依旧成立。v为第k + 1次迭代加入S中的节点，那么v是第k次迭代结束后，T中标记值最小的节点，按照假设，标记值即为从a出发经过S集合中节点到v的最小路径，同时也是a到v的最小路径。如果该标记值不是a到v的最小路径，则一定存在一条a到v的通路，该通路经过不在S集合中的至少一个节点，且通路长度小于$dis_k(v)$。假设p为该通路上不在集合S中的第一个节点，则通路的长度可以表示为$dis_k(p) + d(p, v)$，其中d(p, v)表示p到v的最短路径长度。且以下不等式成立：

由于该算法的使用场景为正权图，故d(p, v)一定大于0，所以一定存在$dis_k(p) < dis_k(v)$，这与$dis_k(v)$为$dis_k(x)$中的最小值矛盾，故$dis_k(v)$即为a到v的最小路径长度。属性1得证。

下面证明v加入S后，更新的标记值为从a出发经过集合S节点的最短路径长度。v加入S后，a到与v相邻节点的只包含S集合中节点的最小通路只有两种变化可能，要么经过v，要么不经过v。所以$dis_{k+1}(q) = \rm {min} (dis_k(q), dis_k(v) + w(v, q))$，其中q表示与v相邻的任意节点，该计算方法就是更新标记值的方法，故属性2得证。

实例

有 n 个网络节点，标记为 1 到 n。

给你一个列表 times，表示信号经过 有向 边的传递时间。 times[i] = (ui, vi, wi)，其中 ui 是源节点，vi 是目标节点， wi是一个信号从源节点传递到目标节点的时间。

现在，从某个节点 K 发出一个信号。需要多久才能使所有节点都收到信号？如果不能使所有节点收到信号，返回 -1 。

分析

本题需要求出从节点K出发到其余各个节点的最短路径中的最大值。延时一定为正值，故为有向正权图，考虑使用Dijkstra算法求解最短路径长度。

代码


class Solution {
    public int networkDelayTime(int[][] times, int n, int k) {
    	// 求解过程中涉及无穷大的加法，为防止越界，设置为最大正整数的一半为正无穷
        final int INF = Integer.MAX_VALUE / 2;
        // 存储图信息
        int[][] g = new int[n][n];
        for (int i = 0; i < n; ++i) {
            Arrays.fill(g[i], INF);
        }
        for (int[] t : times) {
            int x = t[0] - 1, y = t[1] - 1;
            g[x][y] = t[2];
        }
        // 存储标记值
        int[] dist = new int[n];
        Arrays.fill(dist, INF);
        dist[k - 1] = 0;
        boolean[] used = new boolean[n];
        for (int i = 0; i < n; ++i) {
            int x = -1;
            // 找到T集合中的最小标记值
            for (int y = 0; y < n; ++y) {
                if (!used[y] && (x == -1 || dist[y] < dist[x])) {
                    x = y;
                }
            }
            // 最小标记值放入集合S
            used[x] = true;
            // 更新标记值
            for (int y = 0; y < n; ++y) {
                dist[y] = Math.min(dist[y], dist[x] + g[x][y]);
            }
        }

        int ans = Arrays.stream(dist).max().getAsInt();
        return ans == INF ? -1 : ans;
    }
}
